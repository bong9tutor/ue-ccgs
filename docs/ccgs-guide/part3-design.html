시스템 설계 — 게임의 뼈대 만들기

Part 2에서 정리한 각 시스템마다 상세한 Game Design Document(GDD)를 작성합니다

1

/design-system — 시스템별 GDD 작성하기

"GDD"가 뭔가요?

Game Design Document의 약자로, "이 시스템이 정확히 어떻게 작동하는지"를 적은 설계 문서입니다. 건물의 설계도처럼, 만들기 전에 먼저 설계를 합니다. Claude가 대화를 통해 함께 작성해줍니다.

시작하기: MVP 시스템부터

Part 2에서 /map-systems가 정한 우선순위에 따라, MVP 시스템부터 하나씩 설계합니다.

첫 번째 시스템 설계
# 방법 1: 시스템 이름 직접 지정 나 > /design-system combat # 방법 2: 다음 미설계 시스템 자동 선택 나 > /map-systems next

Claude와의 설계 대화 (실제 예시)

Claude가 GDD의 8개 필수 섹션을 하나씩 대화하며 채워나갑니다. 한 번에 전부가 아니라 섹션별로 질문하고 확인합니다.

/design-system combat 대화 예시
Claude (Game Designer): "전투 시스템 GDD를 함께 작성합니다." "8개 섹션을 하나씩 진행할게요." ━━ 섹션 1/8: Overview (개요) ━━ Claude: "이 전투 시스템의 핵심을 한 문장으로 설명하면?" 나 > 실시간으로 콤보를 쌓으면서 적을 쓰러뜨리는 액션 전투 Claude: "좋아요! 이렇게 정리했습니다: '실시간 콤보 기반 액션 전투. 경공격-중공격-스킬의 조합으로 다양한 콤보를 만들고, 적의 패턴을 읽어 회피하며 반격한다.' 맞나요?" 나 > 네! ━━ 섹션 2/8: Core Mechanics (핵심 메커닉) ━━ Claude: "전투에서 플레이어가 할 수 있는 행동들은?" 제가 일반적인 액션 RPG 행동을 제안합니다: - 경공격 (빠른 약한 공격) - 중공격 (느린 강한 공격) - 회피 (구르기/대시) - 방어 (막기/패리) - 스킬 (특수 능력) "추가/제거할 행동이 있나요?" 나 > 방어 대신 패리만 남기고, 잡기 공격도 추가해주세요 Claude: "✓ 업데이트됨. 콤보 시스템은 어떤 느낌이면 좋을까요?" A) 자유 콤보 (아무 순서로 연결) B) 정해진 콤보 루트 (특정 순서로만 발동) C) 혼합 (기본은 자유, 특수 콤보 루트 존재) 나 > C ... (6개 섹션 더 계속 - Balance, Content, Edge Cases 등) ... ━━ 8개 섹션 모두 완료! ━━ Claude: "전투 시스템 GDD가 완성되었습니다!" design/gdd/combat.md 에 작성할까요?" 나 > 네 ✓ design/gdd/combat.md 생성 완료 (8개 섹션)
💡
모르는 건 Claude에게 맡기세요

"잘 모르겠어요, 추천해주세요"라고 말하면 Claude가 전문적인 게임 디자인 지식을 바탕으로 제안해줍니다. 여러분은 "좋아요" 또는 "이건 바꿔주세요"만 말하면 됩니다.

GDD 필수 8개 섹션 안내

#섹션쉽게 말하면
1Overview이 시스템이 뭔지 한 문장 설명
2Core Mechanics플레이어가 할 수 있는 행동들
3Progression시간이 지나면 뭐가 달라지는지
4Balance수치 (데미지, 쿨타임, 비용 등)
5Content실제 콘텐츠 (무기 종류, 스킬 목록 등)
6Technical Constraints기술적 한계 (동시 적 수, 성능 등)
7Edge Cases특수한 상황 (동시 피격, 사망 중 공격 등)
8Dependencies다른 시스템과의 연결 관계

설계 검증하기

개별 GDD 검증: /design-review

각 GDD를 전문가 에이전트가 검토합니다.

GDD 검증
나 > /design-review design/gdd/combat.md Claude: "전투 GDD를 검토합니다..." ✓ 8개 필수 섹션 모두 존재 ✓ 밸런스 수치 정의됨 ⚠ 제안: 패리 타이밍 윈도우를 구체적 프레임으로 명시하면 좋겠습니다 ⚠ 제안: 공중 콤보의 낙하 속도를 정의하세요 "제안 사항을 반영할까요?" 나 > 네, 반영해주세요

전체 GDD 교차 검증: /review-all-gdds

모든 GDD 작성이 끝나면 시스템 간 모순이 없는지 한 번에 검사합니다.

전체 검증
나 > /review-all-gdds Claude: "6개 GDD를 교차 검증합니다..." ✓ combat.md <> health.md: 데미지 공식 일치 ✓ inventory.md <> combat.md: 장비 스탯 연동 일치 ⚠ progression.md <> combat.md: 레벨 스케일링 불일치 발견 → "progression은 최대 레벨 50인데, combat의 스킬은 레벨 100까지" "불일치를 수정할까요?" 나 > 네, 최대 레벨 50으로 통일해주세요
