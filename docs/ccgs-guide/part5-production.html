프로덕션 & 릴리스 — 게임 완성하기

스프린트별로 게임 코드를 구현하고, 테스트하고, 다듬어서 최종 빌드를 만듭니다

1

/dev-story — 스토리 구현 시작하기

드디어 실제 게임 코드를 만듭니다!

이 단계에서 Claude가 UE5.6 C++ 코드와 Blueprint를 직접 작성합니다. 여러분은 코드를 짤 필요가 없습니다 — Claude에게 스토리를 맡기고, 중요한 결정만 하면 됩니다.

자동 전문가 라우팅

Claude는 스토리 내용을 분석해서 적합한 전문가 에이전트에게 자동으로 작업을 배정합니다:

전문가 에이전트담당 분야
ue-gas-specialist전투, 스킬, 버프/디버프 (GAS)
ue-blueprint-specialistUI, 아이템, 이벤트, 게임 로직
ue-replication-specialist멀티플레이, 네트워크 동기화
ue-umg-specialist메뉴, HUD, 위젯 UI
unreal-specialist엔진 전반 (레벨, 카메라, 입력)

구현 예시: 경공격 3타 콤보

스토리 구현
나 > /dev-story ST-002 Claude (ue-gas-specialist): "ST-002 '경공격 3타 콤보' 구현을 시작합니다." "GDD와 아키텍처를 참고해서 구현 계획을 세웠습니다:" ━━ 구현 계획 ━━ 1. UComboComponent 클래스 생성 (콤보 상태 관리) 2. GA_LightAttack (GAS Ability) 생성 3. Enhanced Input에 공격 매핑 추가 4. 애니메이션 몽타주 연결 (3타 각각) 5. 히트 판정 & 데미지 적용 "이 계획으로 진행할까요? 수정할 부분이 있으면 말씀해주세요." 나 > 좋아요, 진행해주세요 ... Claude가 코드를 작성합니다 ... Claude: "파일을 생성했습니다:" ✓ Source/Combat/ComboComponent.h ✓ Source/Combat/ComboComponent.cpp ✓ Source/Combat/Abilities/GA_LightAttack.h ✓ Source/Combat/Abilities/GA_LightAttack.cpp "콤보 타이밍 창에 대해 결정이 필요합니다:" A) 200ms (하드코어 — 빠른 반응 필요) B) 400ms (균형잡힌 — 대부분의 액션 게임 수준) ← 추천 C) 600ms (캐주얼 — 넉넉한 입력 시간) 나 > B Claude: "✓ 타이밍 창 400ms로 설정. 구현 완료!" "에디터에서 플레이 버튼을 눌러 테스트해보세요."
🤖
Claude가 코드를 다 짜줍니다!

C++ 문법을 몰라도 괜찮습니다. Claude가 UE5.6 모범 사례에 맞춰 코드를 작성하고, 여러분은 "A, B, C 중에 뭘 고를지" 같은 게임 디자인 결정만 합니다.

2

/code-review — 코드 리뷰 받기

"코드 리뷰"가 뭔가요?

글을 쓰고 나면 맞춤법 검사를 하듯, 코드도 작성 후 품질 검사를 합니다. Claude가 UE5.6 모범 사례, 성능 문제, 버그를 자동으로 찾아줍니다.

리뷰 실행하기

코드 리뷰
나 > /code-review Claude (Code Reviewer): "최근 변경 사항을 리뷰합니다..." ━━ 리뷰 결과 ━━ ✓ UE5 코딩 규약 PASS 올바른 네이밍 컨벤션 ✓ GAS 패턴 PASS Ability 구조 적절 ✓ 메모리 관리 PASS UPROPERTY 올바르게 사용 ⚠ Null 체크 WARN AnimMontage 포인터 체크 추가 권장 ✓ 성능 PASS Tick에서 불필요한 계산 없음 ✓ 스레드 안전 PASS GameThread에서만 접근 "⚠ 1개 경고: AnimMontage 로드 전 null 체크를 추가하면 에디터 크래시를 방지할 수 있습니다. 수정할까요?" 나 > 네, 수정해주세요 Claude: "✓ null 체크 추가 완료. 모든 항목 PASS!"
3

/story-done — 스토리 완료 처리하기

스토리가 정말 끝났는지 검증합니다

Claude가 GDD에서 정한 수용 기준(Acceptance Criteria)을 하나씩 확인합니다.

스토리 완료
나 > /story-done ST-002 Claude: "ST-002 '경공격 3타 콤보' 수용 기준 검증:" ✓ 공격 버튼 3회 연속 입력 → 3타 콤보 발동 ✓ 타이밍 창(400ms) 밖 입력 → 콤보 리셋 ✓ 각 타격마다 히트 이펙트 재생 ✓ 데미지가 GDD 스펙에 맞게 적용 ✓ 코드 리뷰 통과 결과: PASS — ST-002 완료! → "Done" 상태로 이동
💡
이 3단계가 핵심 루프입니다:

/dev-story → /code-review → /story-done → 다음 스토리로! 이 과정을 스프린트의 모든 스토리에 반복합니다.

4

스프린트 반복 — 게임이 점점 완성됩니다

개발 루프

한 스프린트의 모든 스토리를 완료하면, 다음 스프린트를 시작합니다:

다음 스프린트
# 스프린트 1의 모든 스토리 완료 후 나 > /sprint-plan next Claude: "스프린트 1 완료! 요약:" 완료된 스토리: 5/5 구현된 기능: 플레이어 이동, 카메라, 경공격 콤보, 체력 시스템 "스프린트 2를 계획합니다..." 목표: "적과 싸울 수 있다" 선택 스토리: ST-003(중공격), ST-004(회피), EP5-ST-001(적 AI 기본)...

스프린트를 거듭하면 게임이 성장합니다

스프린트목표완성되는 것
Sprint 1기본 움직임 & 공격걸어다니며 공격하는 캐릭터
Sprint 2적 & 전투적과 싸울 수 있는 게임
Sprint 3UI & 인벤토리체력바, 메뉴, 아이템 관리
Sprint 4레벨 & 보스스테이지와 보스전
Sprint 5저장 & 사운드세이브/로드, BGM, 효과음
Sprint 6폴리시 & 버그 수정완성도 높은 게임!

/perf-profile — 성능 최적화

"성능 최적화"가 뭔가요?

게임이 버벅거리지 않고 부드럽게 돌아가도록 만드는 과정입니다. 60fps(초당 60번 화면 갱신)가 기본 목표입니다. Claude가 UE5.6 전용 성능 분석을 해줍니다.

성능 분석 실행하기

성능 프로파일
나 > /perf-profile Claude: "UE5.6 성능 분석 결과:" ┌─────────────────┬────────┬────────┬──────┐ │ 항목 │ 현재 │ 목표 │ 상태 │ ├─────────────────┼────────┼────────┼──────┤ │ FPS │ 52 │ 60+ │ FAIL │ Lumen GI │ 5.1ms │ <3ms │ FAIL │ Nanite 메시 │ 1.2ms │ <2ms │ PASS │ Draw Calls │ 2100 │ <1500 │ WARN │ 메모리 │ 2.1GB │ <2.5GB │ PASS │ Tick 시간 │ 0.8ms │ <2ms │ PASS └─────────────────┴────────┴────────┴──────┘ "최적화 제안:" 1. Lumen 품질을 High → Medium으로 조정 (2ms 절약) 2. 배경 오브젝트에 HLOD 적용 (Draw Call 30% 감소) 3. 원거리 라이트를 Static으로 변경 "적용할까요?" 나 > 네, 전부 적용해주세요 Claude: "✓ 최적화 적용 완료. FPS: 52 → 64 개선!"

/balance-check — 게임 밸런스 점검

게임이 너무 쉽거나 어렵지 않은지 확인합니다

Claude가 GDD에서 정한 밸런스 수치(데미지, 체력, 경험치 등)가 실제 코드와 일치하는지 검증합니다.

밸런스 체크
나 > /balance-check Claude: "밸런스 검증 결과:" ✓ 플레이어 기본 공격력: 25 (GDD 범위 20-30) ✓ 적 기본 체력: 100 (GDD 범위 80-120) ⚠ 보스 체력: 5000 (GDD 범위 3000-4000) → 너무 높음 ✓ 경험치 곡선: 레벨당 +15% (GDD 범위 10-20%) ✓ 체력 회복 아이템: 50 회복 (GDD: 50) "보스 체력을 3500으로 조정할까요?" 나 > 네 Claude: "✓ 보스 체력 5000 → 3500 수정 완료"

/team-polish — 최종 다듬기

게임에 "느낌"을 더합니다

기능은 다 있지만 화면 흔들림, 타격감, UI 애니메이션, 사운드 효과 같은 "촉감"을 더해야 진짜 게임 같아집니다. Claude가 여러 전문가를 동시에 투입합니다.

팀 폴리시
나 > /team-polish Claude (Director): "폴리시 작업을 전문가들에게 배분합니다:" ━━ 동시 작업 할당 ━━ 🎮 ue-gas-specialist → 타격 시 카메라 쉐이크 & 히트스톱 추가 🎨 ue-umg-specialist → 체력바 감소 애니메이션, 콤보 카운터 이펙트 🎵 unreal-specialist → MetaSounds 타격음, BGM 트랜지션 🐛 code-reviewer → 최종 버그 스캔 ... 각 전문가가 동시에 작업합니다 ... Claude: "폴리시 완료! 변경 사항:" ✓ 카메라 쉐이크 추가 (타격/피격 시) ✓ 히트스톱 60ms (강공격 시) ✓ UI 애니메이션 12개 추가 ✓ 효과음 24개 연결 ✓ 버그 3개 수정
