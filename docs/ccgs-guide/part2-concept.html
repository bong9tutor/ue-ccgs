Part 2 of 5

게임 컨셉 만들기

Claude와 대화하며 게임 아이디어를 발전시키고, 엔진을 설정하고, 게임의 시스템을 정리합니다

1

첫 명령어: /start 입력하기

CCGS의 모든 시작은 /start 입니다

Part 1에서 claude를 실행한 상태에서 아래와 같이 입력합니다.

claude code
나 > /start

Claude가 보여주는 화면

Claude가 프로젝트 상태를 분석한 후 4가지 선택지를 물어봅니다.

claude 응답
Claude: "Claude Code Game Studios에 오신 것을 환영합니다! 지금 게임 아이디어 관점에서 어디쯤 계신가요?" A) 아이디어가 전혀 없어요 B) 모호한 아이디어가 있어요 (장르나 테마 정도) C) 명확한 컨셉이 있어요 D) 이미 만든 작업물이 있어요

어떻게 답하면 되나요?

그냥 자연스럽게 대답하시면 됩니다. 예를 들어:

답변 예시
# 아이디어가 없을 때 나 > A # 모호한 아이디어가 있을 때 나 > B. 로그라이크 느낌의 게임을 만들고 싶어요 # 명확한 아이디어가 있을 때 나 > C. 3인칭 액션 RPG인데, 실시간 전투가 있는 게임이에요

다음으로 Claude가 물어보는 것: 리뷰 모드

리뷰 모드 선택
Claude: "작업하면서 얼마나 많은 검토를 원하시나요?" Full — 매 단계마다 전문가가 검토 (꼼꼼하지만 느림) Lean — 단계 넘어갈 때만 검토 (권장!) Solo — 검토 없이 빠르게 진행 나 > Lean Claude: "좋습니다! Lean 모드로 설정했습니다."
✅
비전공자라면 "Lean"을 추천합니다.

Full은 너무 느리고, Solo는 검증 없이 넘어가서 나중에 문제가 생길 수 있습니다.

2

/brainstorm — 게임 아이디어 만들기

이게 뭘 하는 명령어인가요?

Claude의 Creative Director(창작 감독) 에이전트가 여러분과 대화하며 게임 아이디어를 발전시킵니다. 게임 디자인 전문 기법(MDA 프레임워크 등)을 사용합니다. 아이디어가 없어도 Claude가 질문을 하며 이끌어줍니다.

실행하기

brainstorm 시작
# 아이디어가 없을 때 나 > /brainstorm # 힌트가 있을 때 (아무 말이나 붙여도 됩니다) 나 > /brainstorm 좀비 서바이벌 나 > /brainstorm 귀여운 동물 농장 게임 나 > /brainstorm 우주 탐험 로그라이크

Claude와 대화하는 과정 (실제 예시)

Claude가 차례차례 질문하고, 여러분이 대답하면 됩니다. 정답은 없습니다!

brainstorm 대화 예시 (축약)
Claude: "게임에서 가장 감동받거나 중독된 순간이 있으세요?" 나 > 다크소울에서 보스 잡았을 때 짜릿했어요 Claude: "가장 많이 한 게임 3개는요? 왜 좋아하셨어요?" 나 > 젤다, 엘든링, 하데스. 탐험이랑 전투가 좋아서요 Claude: "좋습니다! 취향 프로필을 정리하면: - 감정: 도전의 짜릿함, 발견의 기쁨 - 선호: 탐험 + 액션전투 + 성장 맞으세요?" 나 > 네 맞아요! Claude: "3가지 완전히 다른 콘셉트를 만들었습니다:" 1. 「Ember Knight」 - 3인칭 액션 RPG "불의 힘을 다루는 기사가 저주받은 세계를 탐험" 핵심 동사: 전투, 탐험, 강화 2. 「Void Drifter」 - 로그라이크 탑다운 슈터 "우주 잔해 사이를 떠돌며 생존하는 1인 파일럿" 핵심 동사: 회피, 수집, 조합 3. 「Stone & Spirit」 - 오픈월드 탐험 "고대 유적의 정령과 교감하며 세계를 복원" 핵심 동사: 탐험, 퍼즐, 교감 "어떤 콘셉트가 끌리세요?" 나 > 1번이 좋아요! Ember Knight로 가겠습니다

이후 Claude가 자동으로 하는 것들

선택한 콘셉트에 대해 Claude가 더 깊이 있게 질문하며 다음을 정의합니다:

항목Claude가 하는 것여러분이 하는 것
코어 루프30초/5분/세션/장기 루프 제안"맞아요" 또는 수정 요청
게임 필라3~5개 핵심 기둥 정의확인 또는 수정
타겟 플레이어Bartle 분류법으로 분석동의 또는 조정
범위(Scope)MVP ~ Full Vision 단계 정의현실적 범위 선택

최종 결과: game-concept.md 파일 생성

brainstorm 완료
Claude: "게임 콘셉트가 완성되었습니다! design/gdd/game-concept.md에 작성할까요?" 나 > 네, 작성해주세요 Claude: "✓ design/gdd/game-concept.md 생성 완료" "다음 단계: /setup-engine 으로 엔진 설정"
3

/setup-engine — 언리얼 엔진 5.6 설정하기

이건 뭘 하는 건가요?

CCGS에게 "우리 게임은 언리얼 엔진 5.6으로 만들 거야"라고 알려주는 것입니다. 이 설정에 따라 이후 모든 에이전트가 UE5.6에 맞는 기술과 패턴을 사용합니다.

실행하기

엔진 설정
나 > /setup-engine unreal 5.6 Claude: "Unreal Engine 5.6 설정을 시작합니다." "이전에 Unreal Engine을 사용해본 적 있으세요?" 나 > 아니요, 처음이에요 Claude: "알겠습니다! 초보자에게 적합한 설정으로 진행합니다." 설정 결과: - 엔진: Unreal Engine 5.6 - 언어: C++ (핵심 로직) + Blueprint (게임 콘텐츠) - 렌더링: Lumen (실시간 글로벌 일루미네이션) - 지오메트리: Nanite (자동 최적화) - 입력: Enhanced Input - UI: CommonUI "CLAUDE.md에 이 설정을 저장할까요?" 나 > 네 ✓ CLAUDE.md 업데이트 완료 ✓ .claude/docs/technical-preferences.md 생성 완료 ✓ docs/engine-reference/unreal/ 레퍼런스 생성 완료 "다음 단계: /map-systems 으로 시스템 분해"
💡
"C++를 모르는데 괜찮나요?"

네, 괜찮습니다! CCGS의 AI 에이전트들이 C++ 코드를 대신 작성합니다. 여러분은 "어떤 기능을 만들지" 결정하고, Claude가 코드를 작성합니다. Blueprint(시각적 프로그래밍)도 함께 사용하므로 코드 없이도 일부 로직을 만들 수 있습니다.

4

/map-systems — 게임 시스템 분해하기

"시스템 분해"가 뭔가요?

게임을 구성하는 기능 단위들을 나누는 것입니다. 예를 들어 액션 RPG는 "전투 시스템", "인벤토리 시스템", "AI 시스템", "세이브 시스템" 등으로 나뉩니다. 이걸 정리해야 하나씩 설계하고 만들 수 있습니다.

실행하기

시스템 분해
나 > /map-systems Claude (Game Designer): "game-concept.md를 분석합니다..." <Ember Knight> 에서 발견한 시스템들: [명시적 시스템 - 게임 콘셉트에서 직접 언급된 것] 1. Combat (전투) — 실시간 콤보, 회피, 스킬 2. Progression (성장) — 레벨, 스탯, 스킬 트리 3. Exploration (탐험) — 맵 이동, 던전, 퀘스트 [암묵적 시스템 - 위 시스템이 작동하려면 필요한 것] 4. Health/Damage — 체력, 데미지 계산 5. Inventory (인벤토리) — 아이템, 장비 6. Enemy AI (적 AI) — 적의 행동 패턴 7. Save/Load (저장) — 게임 진행 상황 저장 8. UI/HUD — 화면에 보이는 정보 9. Input (입력) — 키보드/패드 조작 10. Animation (애니메이션) — 캐릭터 동작 "빠진 시스템이 있나요? 제거할 것은요?" 나 > 사운드 시스템도 추가해주세요. 나머지는 좋아요! ✓ Audio (사운드) 시스템 추가 ✓ 의존성 매핑 완료 ✓ 우선순위 할당 완료: MVP: Input, Health, Combat, Save Alpha: Inventory, AI, Progression, Animation Full: Exploration, Audio, UI/HUD "design/gdd/systems-index.md에 작성할까요?" 나 > 네 ✓ design/gdd/systems-index.md 생성 완료 "다음 단계: /design-system 으로 각 시스템별 GDD 작성"
